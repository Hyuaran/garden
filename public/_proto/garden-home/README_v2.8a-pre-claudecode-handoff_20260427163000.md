## 4. デザインルール

### 4-1. カラーパレット

CSSの `:root` 内で定義。**必ずCSS変数を使うこと**。直接色コードを書かない。

| 用途 | 変数名 | 値 |
|---|---|---|
| 全体背景（紙風） | `--bg-paper` | `#f5f0e1` |
| カード背景（半透明） | `--bg-card` | `rgba(255, 253, 245, 0.75)` |
| メインテキスト | `--text-main` | `#4a4233` |
| サブテキスト | `--text-sub` | `#8a7f6a` |
| 緑アクセント | `--accent-green-d` | `#7a9968` |
| 金アクセント | `--accent-gold` | `#d4a541` |
| 警告 | `--text-warning` | `#c95a4a` |

### 4-2. フォント

**世界観統一のため、すべて明朝・セリフ系で揃える**

| 用途 | フォント | 理由 |
|---|---|---|
| 日本語(本文・見出し) | Noto Serif JP / Shippori Mincho（明朝） | Gardenの世界観（植物・水彩・温かみ）に合う |
| 英字(見出し・モジュール名) | Cormorant Garamond（セリフ体） | 上品でクラシカル、Bloom/Soilなどの単語が美しく見える |
| **数字・通貨・時刻** | **Cormorant Garamond + tabular-nums** | **世界観統一・等幅数字で桁揃え** |

**重要**: 数字には `font-variant-numeric: tabular-nums` + `font-feature-settings: 'tnum', 'lnum'` を必ず指定。これによりカンマや桁が揃い、視覚的なガタつきがなくなる。

**過去の経緯**: v2.1で数字をInterに変更したが、世界観から浮いたためv2.2でCormorant Garamondに戻した（tabular-nums設定で桁揃えは維持）。

**禁止フォント**: ゴシック体は使わない。Inter・Roboto・Arial 等のサンセリフ系も基本的に使わない（数字含めて明朝系で統一）。monospace（等幅）も避ける。

### 4-3. 角丸・影・透明度

- カード角丸: `--radius-card: 18px`
- ガラスっぽさを出すため、`backdrop-filter: blur(6px〜10px)` を多用
- 影は柔らかい茶系（`rgba(120, 100, 70, ...)`）

### 7-1-2. アイコン透明化の重要ルール（過去のトラブル防止）

**新しいアイコンを追加する場合、必ずClaude Chatに透明化処理を依頼すること。**

理由: remove.bg等の無料オンラインツールで透明化したPNGは、見た目は透明に見えても、実際にはRGBモード（透明情報なし）で保存されることがある。この場合、ブラウザ表示時にアイコン背景が黒く表示される深刻な問題が発生する。

**正しい透明化の確認方法**:
1. PNGファイルの「モード」が `RGBA`（4チャンネル）であること
2. 四隅のアルファ値が `0`（完全透明）であること
3. PythonのPILで `Image.open(path).mode` を確認すれば判定可能

**透明化処理スクリプト（参考）**:
```python
from PIL import Image
import numpy as np

def make_transparent(img_path, out_path, tolerance=35):
    img = Image.open(img_path).convert('RGBA')
    arr = np.array(img)
    # 四隅から背景色を推定
    corners = [arr[0:3, 0:3], arr[0:3, -3:], arr[-3:, 0:3], arr[-3:, -3:]]
    bg = np.mean(np.concatenate([c.reshape(-1, c.shape[-1]) for c in corners])[:, :3], axis=0).astype(int)
    rgb = arr[:, :, :3].astype(int)
    diff = np.sqrt(np.sum((rgb - bg) ** 2, axis=2))
    arr[diff < tolerance, 3] = 0
    soft = (diff >= tolerance) & (diff < tolerance * 2)
    soft_alpha = np.clip(((diff - tolerance) / tolerance) * 255, 0, 255).astype(np.uint8)
    arr[soft, 3] = soft_alpha[soft]
    Image.fromarray(arr, 'RGBA').save(out_path, 'PNG')
```

## 8. デザイン上の判断メモ

実装時に意図を汲んでほしいポイント。

### 8-1. なぜ明朝体か
ゴシック体は「業務的・冷たい」印象。Gardenの世界観（植物・水彩・温かみ）には明朝体が合う。Cormorant Garamondはクラシカルだが古臭くなく、数値を表示すると上品に見える。

### 8-2. なぜ半透明・blurを多用するか
ガラス玉アイコンの世界観に合わせ、UI全体を「ガラスや水を通して見る」イメージにしたかったため。背景画像（6種類）を切り替えても、UIが背景と喧嘩しない構造になっている。

### 8-3. なぜ12個のアイコンを4×3で配置か
- 黄金比に近い縦横比でバランスが良い
- 一画面に収まる
- 横スクロールが発生しない

### 8-4. アニメーション
ロード時のフェードインのみ。過剰なホバーアニメーションは業務効率を下げるため避ける。

| 順位 | 区分名 | 状態 | 想定役割 |
|---|---|---|---|

- ヘッダーのユーザーエリアには「**雇用形態 / 権限区分**」の形式で表示
- 例: 「正社員 / 全権管理者」「アルバイト / トス」「正社員 / クローザー」
- 雇用形態と権限区分は完全に独立（アルバイトでも責任者権限を持つことはあり得る）

- Supabaseの `users` テーブルには `employment_type`（雇用形態）と `permission_level`（権限区分）の2カラムを必ず設ける
- 権限区分の (CS)・(一般社員) は仮名のため、`permission_level` は数値ID（1〜7）で管理し、表示名は別マスタで管理することを推奨
- 表示名マスタを別管理することで、東海林さんが正式名称を決めた時に、コード修正なしで反映できる
