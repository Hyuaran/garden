## 投下用短文（東海林さんが各セッションにコピペ）

宛先部分「`a-XXXX`」を該当セッション名に置き換えてコピペしてください。例: `a-bloom` 宛なら 2 箇所書き換え。

~~~
⭐ main-1
【a-main-010 から a-XXXX への 横断指示（dispatch ヘッダー形式 ルール改訂 v3）】
発信日時: 2026-05-01(金) 23:45

dispatch / セッション間メッセージのヘッダー形式を恒久ルール化しました。
今後の発信ではこの形式に統一してください。

【標準ヘッダー形式（3 行構成）】

1 行目: <アイコン> <接頭辞>-NNN
2 行目: 【<発信元> から <宛先> への <種別>（任意で 件名）】
3 行目: 発信日時 YYYY-MM-DD(曜) HH:MM
4 行目: 空行
5 行目以降: 本文

【発信日時の詳細フォーマット】

- 日付: YYYY-MM-DD（例: 2026-05-01）
- 曜日: (月)(火)(水)(木)(金)(土)(日)（例: (金)）
- 時刻: HH:MM（24 時間表記、例: 23:45）
- 完成形: 半角スペース連結（例: 2026-05-01(金) 23:45）

→ 曜日と時刻まで必須。後追い時系列追跡のため。

【接頭辞ルール（略さない）】

| セッション | 接頭辞 |
| a-main / a-main-005-010 等 | main（並列ワークツリー共通カウンター、引き継ぎ）|
| a-soil | soil |
| a-root | root |
| a-root-002 | root-002（並列は別カウンター）|
| a-tree | tree |
| a-leaf | leaf |
| a-leaf-002 | leaf-002 |
| a-bud | bud |
| a-bloom | bloom |
| a-bloom-002 | bloom-002 |
| a-seed | seed |
| a-forest | forest |
| a-rill | rill |
| a-auto / a-auto-002 / a-auto-004 | auto / auto-002 / auto-004 |
| a-review | review |
| b-main | b-main |
| 作業日報セッション | daily-report |

→ 略すと混乱するため必ず略さない。

【番号付与ルール】

- 各セッションが独自カウンター: C:\garden\a-XXXX\docs\dispatch-counter.txt
- 2026-05-01 から各セッション 1 開始（過去発信は遡及対応せず）
- リセットしない（同系統セッションをまたいで通し番号、引き継ぎ書で次番号引き継ぎ）

【番号付与パターン】

- a-main → モジュール: main カウンター（main-114）
- モジュール → a-main: モジュール側カウンター（bloom-15, tree-7 等）
- モジュール間: 発信元カウンター（a-tree → a-bloom なら tree-7）
- a-auto → 複数モジュール: auto カウンター（auto-23、宛先別ではなく 1 つの番号）

【重要度アイコン（1 行目の数字の前に必ず付与）】

- 🔴 緊急: 即対応必要
- ⭐ 重要 / 優先: 当日中の対応推奨
- 🟢 通常: 通常優先度
- 🟡 中: 緊急ではないが対応必要
- 📋 確認のみ: アクション不要、情報共有のみ

【種別の語彙】

dispatch / 受領確認 / 進捗報告 / 完了報告 / 質問 / 確認 / 共有 /
横断指示 / 周知 / 引き継ぎ

【適用範囲】

- 他セッション向けの投下短文・dispatch・回答すべて
- 東海林さんがコピペで配布するすべてのメッセージ
- a-XXXX から a-main / 他モジュール宛の発信も対象

【ファイル名にも番号付与】

dispatch-<接頭辞>-<番号>-<内容>-YYYYMMDD.md
例: dispatch-main-114-bloom-002-v2-8a-decisions-20260501.md

【引き継ぎ方法（コンテキスト増加時）】

1. セッション 80% 到達 → ハンドオフ書（handoff-XXX-to-YYY.md）作成
2. ハンドオフ書末尾に「dispatch counter: 次番号 NNN」記載
3. 新セッション作成時、docs/dispatch-counter.txt に NNN 書き込み
4. 新セッション起動 → 同接頭辞で連続採番

例: a-bloom 終了時 bloom-200 → a-bloom-002 引き継ぎ → bloom-002-1 から開始
例: a-main-010 終了時 main-200 → a-main-011 引き継ぎ → main-201 から再開

【東海林さん略応答パターン】

東海林さんは「114 OK」「main-114 OK」のように略応答することが多い。
受信側が略形で文脈推定不能な場合、必ず東海林さんに質問してください。

【既存運用との整合】

ヘッダーは ~~~ or ``` のコードブロック内の冒頭 3 行に必ず配置。
ファイル化 + markdown link 案内のルールはそのまま継続。

【背景】

2026-05-01(金) 東海林さんから明示指示：
