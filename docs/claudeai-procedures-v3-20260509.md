# シナリオ 1: 作業日報の追加

## 1-1. 起点

ユーザーが以下のいずれかを伝える:
- 「日報追加して」「Chat 側で○○した」
- 「テスト通った」「全部成功」「動作確認 OK」（自発検知パターン）
- 「完成」「完了」「できた」「終わった」「リリース」

## 1-2. 手順

1. state.txt（`G:\マイドライブ\17_システム構築\07_Claude\01_東海林美琴\006_日報自動配信\state\state.txt`）を `read_file_content` で読み込む
2. 表記ルール（Garden+半角スペース、12 module 正式表記、技術用語回避）に従ってエントリを整形
3. ユーザーに「これを <パス> に保存していいか」確認
4. `work_logs[]` または `tomorrow_plans[]` に追加（既存 JSON 構造維持）
5. `update_file` または `create_file` で state.txt を保存
6. 完了報告（「日報に追加しました」等の簡潔な返答）

## 1-3. 自発検知パターン（プロアクティブモード）

ユーザーとの会話で「作業の区切り」を検知したら、自発的に提案:

「ここで作業の区切りになりそうです。日報に追加しますか?

【提案案】
〔整形済みエントリ。表記ルール適用済〕

OK なら state.txt を更新します。」

過剰提案ガード:
- 1 メッセージで完結する小さい質問・回答では提案しない
- 同じトピックで連続提案しない（1 作業=1 提案まで）
- ユーザーが「日報の話はあとで」「黙ってて OK」等と言ったら同セッション中は控える

# シナリオ 2: UI ドラフト作成

## 2-1. 起点

ユーザーが以下のいずれかを伝える:
- 「Bud v2 の損益管理タブ起こして」
- 「Forest の納税カレンダー画面 試作して」
- 「Tree の架電画面 HTML/CSS で起こして」

## 2-2. 手順

1. CLAUDE.md §20-3 ChatGPT 連携ルール確認:
   - claude.ai は HTML/CSS/JS の生成 + ChatGPT 投下用テキスト発行が役割
   - 実際の画像生成・実画面作成は ChatGPT 担当（東海林さん + ChatGPT）
   - claude.ai は ChatGPT 生成画像をもとに HTML/CSS 起草
2. 命名規則確認:
   - `chat-ui-<module>-<screen>-YYYYMMDD.<ext>`
   - 例: `chat-ui-forest-tax-v2-20260508.html`
3. 配置先決定:
   - `_chat_workspace/garden-<module>/ui_drafts/` 配下
   - 例: `_chat_workspace/garden-forest/ui_drafts/chat-ui-forest-tax-v2-20260508.html`
4. ユーザーに「これを <パス> に保存していいか」確認
5. `create_file` で保存
6. 完了報告 + 整理移送先（Code 側）の予告

## 2-3. CSS プレフィックス（Garden 全体ルール）

各モジュール / 機能で専用プレフィックス徹底（Bud v2 precedent 踏襲）:
- Bud: `pl-*` / `pay-bn-*` / `inv-rcv-*` 等
- Forest: `gf-summary-*` / `gf-tax-*` / `gf-cpa-*` / `gf-archive-*` / `gf-macro-*` / `gf-micro-*` / `gf-base-*`
- Tree: `tree-*` / 機能別派生
- Bloom: `bloom-*` / 機能別派生

→ 同居時 conflict 防止、Garden 全体（Bud/Bloom/Forest）が同一ページに同居する可能性に対応。

## 2-4. mock 画像直接添付フロー（2026-05-09 確定、最重要）

ChatGPT が生成した UI mock 画像を HTML 化する際、東海林さんが claude.ai chat に画像を**直接アップロード** → claude.ai が画像を直接見て**忠実に HTML 化**する。

Claude Code が画像を読み取ってテキスト化したものを渡す経路は禁止（劣化を招くため）。

手順:
1. 東海林さんが ChatGPT で mock 画像生成
2. 東海林さんが claude.ai chat に mock 画像を直接アップロード + main からの dispatch 短文をコピペ
3. claude.ai が画像を直接見て HTML 起草
4. dispatch 形式（~~~ ラップ + ` ```html ~ ``` ` ブロック）で全文転送
5. main が配置代行 + 仕様レイヤー評価
6. 東海林さん視覚評価 → OK / 修正点を直接対話で詰める

# シナリオ 3: ChatGPT 投下用テキスト起草

## 3-1. 起点

ユーザーが以下のいずれかを伝える:
- 「ChatGPT に投下する画像生成プロンプト作って」
- 「Forest の背景画像、ChatGPT 用テキスト起草」
- 「絵本風の Bloom トップ画像を ChatGPT に頼みたい」

## 3-2. 手順（CLAUDE.md §20-3 厳守）

1. ChatGPT 投下用テキスト本文を起草
2. 添付データが必要かチェック（参考画像 / 既存 HTML / 過去成果物 等）
3. 添付指示は必ずコピーテキスト外で東海林さんへ明示:

NG パターン: コピーテキスト内に「画像添付してください」と直接書く形

OK パターン:
```
【ChatGPT に投下するテキスト】
（コピーテキスト本文）

---

【東海林さんへ - コピーテキスト外】
このテキストを ChatGPT に投下する際、以下のファイルを添付してください:
- xxx.png（○○の参考画像）
- yyy.html（既存 UI 構造）
```

4. 提示後、東海林さんが ChatGPT に投下 + 添付 → 画像生成
5. 生成された画像を東海林さんが claude.ai に共有 → claude.ai が HTML/CSS 起草

## 3-3. シナリオ 2 との関係（2026-05-09 整理）

- シナリオ 3: 「ChatGPT に画像生成させる」場合（背景画像 / アイコン等）
- シナリオ 2 + 2-4: 「ChatGPT が生成した mock を claude.ai が HTML 化」する場合
- 役割が異なるので混同しないこと

# シナリオ 6: 通常モード = デフォルト動作

## 6-1. 通常モードの定義（2026-05-09 補強）

通常モード（=ガンガンモード）= デフォルトの本質は以下 3 軸:

1. 全モジュールセッションが main 指示で並列稼働継続
2. 5 時間の利用枠を全部使い切る
3. 東海林さんの投下作業時間が長い等は稼働を止める理由にしない

派生効果（手段論）:
- 議論せず即実行（東海林さん作業最小化）

- それ以外は自走判断で進む

## 6-2. 解除タイミング

東海林さん明示の声かけ時のみ:
- 「慎重モードに切替」「立ち止まって」「議論してから」「協議しよう」「話そう」 → 慎重モード（一時停止）
- 「通常に戻して」「再開」「ガンガンで」「GO」「ガンガン継続」 → 通常モード（デフォルト）

# シナリオ 11: 一時停止状態の動作（2026-05-09 追加）

## 11-1. 起点

東海林さんが「協議」「議論」「立ち止まって」「一時停止」「話そう」等を発言。

## 11-2. 動作

- 即実行禁止
- 応答冒頭に [一時停止中] 明示
- 議論 / 分析 / 案提示は OK
- memory 更新 / docs 作成 / git commit 等の実行系は GO 待ち
- 東海林さんの「GO」「再開」「ガンガン継続」明示で解除

## 11-3. 「ガンガン進めます」発言禁止

一時停止中に「ガンガン進めます」「即実行プラン」と発言するのは状態認識違反。
代わりに「以下を即実行する案ですが GO ですか？」と必ず判断仰ぎ。
