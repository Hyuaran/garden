## 1. リハーサルの目的

5/5 後道さんデモは **5-10 分の対面 + 直感判断引き出し** の超圧縮設計。
本番で東海林さんが躊躇なく操作できるよう、**4-5 回練習**して以下を体に染み込ませる：

| 要素 | 目標 |
|---|---|
| キーボード操作 | 1 / 2 / ... / 6 / A / ←→ をスムーズに、画面見ずに押せる |
| atmosphere 解説 1 文 | 6 atmospheres それぞれを 1 文で言える |
| Phase 別タイミング | 6 分構成のリズム感をつかむ |
| 質問対応 | Q&A 7 件にすぐ反応できる |
| 反応観察 | 後道さんの表情・言葉に集中できる |
| トラブル対応 | dev server 落ちた時の即復旧手順 |

## 3. 6 分構成の練習スクリプト

### Phase 1: つかみ（0:00-0:30、目標 30 秒）

**東海林さんの言葉（1 字 1 句覚えなくて OK、雰囲気で）**:
```
「後道さん、ちょっと見てもらいたいものがあるんです。
 Garden の画面、こんな感じになってきました。
 一緒に見てもらえますか？」
```

**操作**:
- ノート PC を後道さんが見える位置に向ける（既に開いてる）
- 既に atmosphere 1 watercolor-tree が表示されてる状態
- 後道さんが画面見るのを確認

### Phase 2: カルーセル開始（0:30-1:00、目標 30 秒）

**東海林さんの言葉**:
```
「Garden は時間帯や業務内容に応じて、画面の雰囲気が変わるんです。
 ちょっと見ててください...」
```

**操作**:
- キー `A` を押す（auto モード開始、画面が 8 秒ごとに切り替わり始める）
- 切り替わり始めたら 1-2 秒様子を見る
- 後道さんの第一印象を観察（表情）

### Phase 3: 6 atmospheres 一周（1:00-2:30、目標 90 秒）

**6 atmospheres 解説（各 1 文、カルーセル進行に合わせて）**:

| atmosphere | 自然な解説（東海林さんの言葉）|
|---|---|
| 1. watercolor-tree | 「これがニュートラル、起動直後の atmosphere」 |
| 2. morning-calm | 「朝もや、ログイン直後の落ち着いた atmosphere」 |
| 3. dynamic-energy | 「業務時間帯、活力のある atmosphere」 |
| 4. analytical-precision | 「データ分析・決算時の透明感ある atmosphere」 |
| 5. system-sync | 「モジュール連携、データが流れる atmosphere」 |
| 6. workflow-flow | 「Chatwork 連携、コミュニケーション活発な atmosphere」 |

→ 8 秒間隔で切り替わるので、解説は **5-6 秒以内**で言い切る。
→ 全 6 で 6×8s = 48s + 解説間 = 約 90 秒。1 周回ったら次 Phase。

### Phase 4: 反応引き出し（2:30-4:30、目標 120 秒）

**東海林さんの言葉**:
```
「どれが一番気に入りましたか？」
```

**操作 + 対応**:

→ 後道さんが答える前に **沈黙を待つ**（最重要）。

**反応パターン別対応**:

| パターン | 対応 |
|---|---|
| A: 1 つを明確に気に入る | キー `<番号>` でその atmosphere に戻る → 「ありがとうございます、この方向で進めますね」 |
| B: 2 つで迷う | 両方戻して比較 → 「両方の良いところ取りもできるか考えます」 |
| C: 全部 NG | 「分かりました、別の atmosphere も用意できます」 |
| D: 全部いい | 「ありがとうございます、業務に応じて切替える方向で進めます」 |
| E: 質問が先行 | Phase 6 へ早めに移行 |

### Phase 5: 12 アイコン演出（4:30-5:30、目標 60 秒）

**東海林さんの言葉**:
```
「あと、Garden は触ると応えるんです、ちょっと見てください」
```

**操作**:
- キー `2`（morning-calm）で落ち着いた atmosphere に戻る
- マウスをモジュールアイコンにホバー（hover 演出を 1-2 個見せる）
  - 推奨: Tree（葉揺れ）、Rill（川波紋）、Calendar（月瞬き）
- 各 hover で 2-3 秒見せる

**東海林さんの言葉**:
```
「12 のモジュールそれぞれに固有の演出があります。
 業務に応じて、画面が呼吸するように変わるんです。」
```

→ 後道さんの反応観察、笑顔出れば成功。

## 4. リハーサル手順（東海林さん 1 人で）

### 推奨頻度

| 日 | 練習回数 | 目的 |
|---|---|---|
| 4/28 | 1 回 | 全体感を掴む、6 分構成のリズム確認 |
| 4/30 | 2 回 | キー操作スムーズ化、6 atmosphere 解説暗記 |
| 5/2 | 2 回 | 反応パターン 5 種の対応練習、Q&A 暗記 |
| 5/4（前日）| 1-2 回 | 通しリハ、本番想定で全力 |

### 各回の流れ

1. PC 開く、http://localhost:3002 を全画面
2. タイマー（スマホ等）で 6 分計測
3. Phase 1-6 を声に出して進行
4. 鏡 or PC のカメラで自分の表情も確認
5. 終わったら以下を振り返り：
   - 6 分以内に収まったか
   - 解説 1 文が 5-6 秒に収まっていたか
   - キー操作で詰まった瞬間があったか
   - 反応パターン分岐がスムーズにできたか
   - Q&A 7 件で詰まったものはあったか
6. 改善点メモ

## 5. トラブル対応（リハーサル中 / 本番）

### dev server 落ちた

```
Step 1: 別ターミナルで http://localhost:3002 を再起動
Step 2: 起動 30 秒以内に復旧不可なら → バックアップ動画スクショ表示
Step 3: 動画スクショで Phase 2-5 を進行（カルーセル切替は手動操作不可、流れだけ見せる）
```

### キーボードショートカット動かない

```
Step 1: F11 でフルスクリーン解除 → 再度 F11
Step 2: それでもダメなら → 画面端の「次へ」ボタンクリックで手動切替
Step 3: 「キーボードショートカットがあります」と説明だけ
```

### 後道さんが想定外の質問をする

```
Step 1: 焦らず「いい質問ですね」で時間を稼ぐ
Step 2: 該当機能をマウスで指して説明
Step 3: 答えられない場合「持ち帰って調べて、明日お返事します」と素直に
```

### 後道さんが「全部見たくない、結論だけ教えて」

```
Step 1: 短いパターン（4 分）に切替、Phase 1-3 簡略化
Step 2: 「2 つの方向性、どっちが良さそうですか？」と直接聞く
Step 3: 反応観察、それで終わり
```
