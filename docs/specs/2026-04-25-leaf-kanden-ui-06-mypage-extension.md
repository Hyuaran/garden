# Leaf 関電 UI #06: マイページ拡張（6 項目追加）

- 対象: Leaf 関電のマイページに業務支援情報 6 項目を追加
- 優先度: **🟡 高**（現場担当の日常業務支援、検針業務効率化）
- 見積: **1.2d 実装** / 0.2d spec
- 担当セッション: a-leaf
- 作成: 2026-04-25（a-auto 002 / Batch 13 Leaf 関電 UI #06）
- 前提:
  - spec-cross-ui-01（カード形式 + ダーク/ライト）
  - spec-cross-storage（Batch 7、JSON 静的データの管理）

---

## 1. 目的とスコープ

### 目的

現場担当者が業務中に頻繁にアクセスする 6 種類の情報をマイページに集約し、外部ツール（紙資料・別ブラウザ）への往復をゼロに近づける。

### 含める

- 6 項目の追加:
  1. 検針カレンダー
  2. 燃調費一覧
  3. 市場価格
  4. 関電チャットリンク
  5. 関電 FD（資料リンク集）
  6. 他社電力会社の客版羅列一覧
- 各項目をボタン化、スクロール可能カード形式
- データソース別の更新方針（外部 API / 静的 JSON / 手動更新）
- データ管理 admin UI

### 含めない

- 個人情報の編集（既存 Root マイページ）
- ログイン関連設定（#05）

```json
  "regions": [
    {
      "region": "大阪市北区",
      "meter_reading_dates": ["1/15", "2/15", "3/14", "4/15", ...]

      "region": "茨木市",
      "meter_reading_dates": ["1/20", "2/19", "3/19", "4/19", ...]

  ]
```

### 2.5 ⑤ 関電 FD（資料）

#### 用途

関電キャリアから提供される業務マニュアル・最新通達等の資料リンク集

#### データ

```sql
CREATE TABLE bud_kanden_fd_resources (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  category        text NOT NULL,    -- '業務マニュアル' / '通達' / '料金表' 等
  title           text NOT NULL,
  url             text,
  storage_key     text,             -- PDF を Storage に保存する場合
  description     text,
  effective_date  date,
  expires_date    date,
  uploaded_by     text REFERENCES root_employees(employee_number),
  created_at      timestamptz NOT NULL DEFAULT now(),
  deleted_at      timestamptz
);
```

#### UI

```
┌──────────────────────────────────────┐
│ 関電 FD（資料）                       │
│                                      │

│    [PDF を開く] [DL]                  │
│                                      │
│ 📄 関電通達 #2026-12 (2026-04-15)     │
│    [PDF を開く]                       │
│                                      │
│ 📄 料金表 2026 年 4 月 (2026-04-01)   │
```

- admin が資料を Storage アップロード or 外部リンクを登録

## 6. RLS

### 6.1 マスタテーブル

```sql
ALTER TABLE bud_kanden_fd_resources ENABLE ROW LEVEL SECURITY;

-- 全 kanden 業務ユーザーが SELECT 可
CREATE POLICY bkfr_select_all ON bud_kanden_fd_resources FOR SELECT
  USING (leaf_user_in_business('kanden') AND is_user_active());

-- 編集は admin+ のみ
CREATE POLICY bkfr_all_admin ON bud_kanden_fd_resources FOR ALL
  USING (has_role_at_least('admin'));
```

その他テーブル（calendar / fuel cost / competitor）も同様。

### 6.2 PDF Storage

- bucket `bud-kanden-fd-pdfs`
- 読み取りは kanden 業務ユーザー全員
- アップロードは admin+

## 7. キャッシュ戦略

### 7.1 静的データ

- 検針カレンダー / FD 一覧: クライアント側 24h キャッシュ
- 市場価格: 1 時間キャッシュ
- 燃調費: 1 日キャッシュ

### 7.2 React Query 等の利用

```ts
const { data: calendar } = useQuery({
  queryKey: ['kanden-calendar', fiscalYear],
  queryFn: fetchCalendar,
  staleTime: 24 * 60 * 60 * 1000,  // 24h
});
```

## 8. 実装ステップ

1. **Step 1**: 4 つの新規テーブル（calendar / fuel / competitor / fd）migration + RLS（1.5h）
2. **Step 2**: マイページの「業務支援情報」セクション（カード形式）（1.5h）
3. **Step 3**: 6 項目の詳細表示モーダル（2h、平均 20min/項目）
4. **Step 4**: admin データ管理画面（タブ切替）（2h）
5. **Step 5**: Excel 取込（共通 CsvImporter 利用）（1h）
6. **Step 6**: キャッシュ戦略（React Query）（0.5h）
7. **Step 7**: モバイル対応（カードグリッド + モーダル）（0.5h）
8. **Step 8**: 結合テスト（1h）

**合計**: 約 **1.2d**（約 10h）

## 10. 判断保留事項

- **判1: JEPX 連携の自動化**
  - Cron 取込 / 手動更新
  - **推定スタンス**: Phase 1 手動、Phase 2 Cron（CSV 取得 + parse）
- **判2: 検針カレンダーの取込形式**
  - Excel / CSV / 手入力
  - **推定スタンス**: Excel 取込（既存資料が Excel 中心）
- **判3: FD PDF の保存場所**
  - Garden Storage / 関電が提供する URL のみ
  - **推定スタンス**: 両方対応（local PDF + 外部 URL）
- **判4: 他社電力データの収集自動化**
  - スクレイピング / 手動 / 外部 API
  - **推定スタンス**: 手動（公開情報更新頻度低）
- **判5: 市場価格の表示粒度**
  - 30 分単位 / 1 時間単位 / 平均値のみ
  - **推定スタンス**: 30 分単位（JEPX 標準）+ サマリ値
- **判6: 6 項目の表示順**
  - 利用頻度順 / アルファベット / カテゴリ
  - **推定スタンス**: カテゴリ順（カレンダー / 価格 / 連絡 / 資料）
- **判7: ユーザーごとカスタマイズ**
  - 表示・非表示 / 並び替え
  - **推定スタンス**: Phase 1 は固定、Phase 2 でカスタム
