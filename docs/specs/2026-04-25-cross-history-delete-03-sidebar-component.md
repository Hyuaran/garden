- 対象: 全モジュール詳細ページ右側の履歴パネル React コンポーネント
- 優先度: **🔴 最高**（UX の中核、既存 UI への組込必須）
- 見積: **0.4d**
- 担当セッション: a-main + 各モジュール
- 作成: 2026-04-25（a-auto 002 / Batch 14 Cross History #03）
- 前提:
  - Cross History #01（テーブル）/ #02（RLS）
  - spec-cross-ui-01（Header / ThemeProvider）

## 1. 目的とスコープ

### 目的

Kintone 風の「レコード履歴」パネルを Garden 全アプリの詳細ページ右側に**収納コンポーネント**として実装、開閉ボタンで表示制御、フィルタ・ページング・差分表示で快適な UX を提供。

### 含める

- React コンポーネント `<ChangeHistorySidebar>` 設計
- 開閉ボタン + アニメーション
- タイムライン表示（時系列、新→古）
- フィルタ（期間 / 操作タイプ / ユーザー）
- ページング（無限スクロール）
- 変更前後の diff 表示
- 削除済・復元の視覚化
- モバイル対応（フルスクリーンモーダル化）

### 含めない

- データ取得 API 詳細（#01/#02）
- 各モジュール統合（#05）
- テスト（#06）

## 2. コンポーネント API

### 2.1 Props

```tsx
type ChangeHistorySidebarProps = {
  module: AppKey;            // 'leaf' | 'bud' | ...
  tableName: string;         // 'soil_kanden_cases' 等
  recordId: string;          // レコード ID
  recordTitle?: string;      // 「案件 #001 山田太郎」等、ヘッダ表示用
```

## 6. ページング・ロード

### 6.1 初回ロード

最新 50 件をロード、それより古い履歴は「もっと見る」ボタンで追加ロード。

### 6.2 無限スクロール（オプション）

```ts
const { ref, inView } = useInView();
useEffect(() => {
  if (inView && hasMore) loadMore();
}, [inView]);
```

スクロール末端で自動的に追加ロード。Phase 1 はボタン式、Phase 2 で無限スクロール検討。

### 6.3 取得 API

```ts
async function fetchHistory({
  module, tableName, recordId,
  limit = 50, offset = 0,
  filters
}: HistoryQuery): Promise<HistoryItem[]> {
  return supabase.from('garden_change_history')
    .select('*, root_employees!changed_by(name)')
    .eq('module', module)
    .eq('table_name', tableName)
    .eq('record_id', recordId)
    .gte('changed_at', filters.from)
    .lte('changed_at', filters.to)
    .in('operation', filters.operations)
    .order('changed_at', { ascending: false })
    .limit(limit)
    .range(offset, offset + limit - 1);
}
```

## 7. 差分表示（diff 風）

### 7.1 単純テキスト

```
旧: 山田太朗
新: 山田太郎
```

色分け: 旧 = 薄赤背景、新 = 薄緑背景

### 7.2 長文 diff

500 文字超の text 列は **行単位 diff**:

```
- 第一段落...
+ 第一段落（修正後）...
  第二段落（変更なし）
- 第三段落...
+ 第三段落（修正後）...
```

ライブラリ: `diff` (npm)

### 7.3 数値・日付

```
金額: ¥10,000 → ¥12,000 (+20%)
期日: 2026-05-01 → 2026-05-15 (+14 日)
```

### 7.4 JSON / 配列

```
オプション: { "color": "red", "size": "M" }
       → { "color": "blue", "size": "M" }
変更: color のみ
```

ネストした JSON は深さ 3 まで diff、それ以上は折りたたみ。

## 9. パフォーマンス

### 9.1 目標

| 指標 | 目標 |
|---|---|
| 初回 50 件取得 | < 500ms |
| 「もっと見る」追加 50 件 | < 300ms |
| フィルタ変更時の再取得 | < 500ms |
| 全文検索（300 件中）| < 200ms |
| diff 計算（500 文字）| < 50ms |

### 9.2 最適化

- **React Query** で自動キャッシュ（1 分有効）
- ユーザー一覧は別クエリで取得（重複削減）
- 開いている時のみクエリ発行（閉じている時は queryDisabled）

## 10. アクセシビリティ

### 10.1 ARIA

- `<aside aria-label="変更履歴">`
- 開閉ボタン: `aria-expanded="true|false"`、`aria-controls="history-panel"`
- 各履歴項目: `<article>` + `aria-label="2026-04-25 14:30 山田太郎による更新"`
- フィルタ: `<form aria-label="履歴フィルタ">`

### 10.2 キーボード操作

- `Tab` で履歴項目間移動
- `Enter` で詳細展開
- `Esc` で詳細閉じる
- `Ctrl + H` で履歴パネル開閉（モジュール内ショートカット）

### 10.3 prefers-reduced-motion

- 開閉アニメーションを無効化
- 即時切替

## 13. 判断保留事項

- **判1: 開閉ボタンの位置**
  - 右側固定 / 詳細画面内 / フローティング
  - **推定スタンス**: 右側固定（Kintone 踏襲）
- **判2: 初期状態**
  - 開いている / 閉じている
  - **推定スタンス**: 閉じている（情報量多、邪魔にならない）
- **判3: ページング方式**
  - ボタン式 / 無限スクロール / 完全ロード
  - **推定スタンス**: Phase 1 ボタン、Phase 2 無限スクロール
- **判4: 詳細展開の方式**
  - インライン展開 / モーダル
  - **推定スタンス**: モーダル（多列変更時の見やすさ）
- **判5: diff ライブラリ**
  - `diff` / 自前実装 / `react-diff-viewer`
  - **推定スタンス**: `diff` + 自前 React コンポーネント
- **判6: フィルタの永続化**
  - localStorage / なし
  - **推定スタンス**: localStorage（最後のフィルタ状態保持）
- **判7: 履歴 0 件時の表示**
  - 「履歴なし」 / パネル非表示
  - **推定スタンス**: 「履歴なし」表示（パネル存在を示す）
- **判8: モバイルの開閉ボタン**
  - フローティング / ヘッダー / メニュー
  - **推定スタンス**: フローティング（右下、片手で届きやすい）
