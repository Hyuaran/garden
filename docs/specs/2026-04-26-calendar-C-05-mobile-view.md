# Calendar C-05: スマホ対応閲覧モード（staff 以上限定 / view-only）

- 対象: スマホ・タブレットでの Calendar 閲覧モード（編集は PC のみ）
- 優先度: 🟡
- 見積: **1.50d**（0.25d 刻み）
- 担当セッション: a-calendar（実装）/ a-root（権限）/ a-bloom（レビュー）
- 作成: 2026-04-26（a-auto 006 / Batch 18 Calendar C-05）
- 前提:
  - **Sprout v0.2 spec §15**（Calendar 設計）
  - **Calendar C-01 / C-02**（基盤完了）
  - 既存 Garden の「社内 PC 限定」アクセス制御の例外を新設

---

## 1. 目的とスコープ

### 1.1 目的
- staff 以上が外出先・移動中にスマホで Calendar を**閲覧のみ**できる UX を提供する
- 既存 Garden の「社内 PC 限定」セキュリティ要件の**Calendar に限った例外**を設計
- スマホでは編集・新規作成を不可とし、誤操作・盗み見を最小化
- 縦持ち / 横持ち両対応のレスポンシブ design

### 1.2 含めるもの
- スマホ判定（User-Agent + viewport width）と専用レイアウト
- 月 / 週 / 日 / リスト 4 ビューの簡易版
- 自分の予定の閲覧
- 参加者として招待された予定の閲覧
- イベント詳細表示（タイトル / 時間 / 場所 / 参加者）
- 「PC で編集する」リンク（PC 案内）

### 1.3 含めないもの
- スマホでの予定作成 / 編集 / 削除
- 通知のプッシュ受信（C-06 で扱う）
- 画面共有・スクリーンショット制御
- ダークモード（v2）
- オフラインキャッシュ（v2）

---

## 2. 設計方針 / 前提

- **既存ルールの例外宣言**: Garden 全モジュール「社内 PC 限定」だが、Calendar のみ staff+ にスマホ閲覧を許可（admin 判断記録：本 spec § 10 で要承認）
- **デバイス判定**: User-Agent + `window.matchMedia('(max-width: 768px)')` の二重判定
- **権限**: Calendar C-02 と同じ（toss / closer / cs はスマホでもアクセス不可）
- **編集禁止の徹底**: API レイヤで User-Agent がモバイルかつ書き込み系エンドポイントを叩いた場合 403
- **デザインベース**: Tailwind モバイルファースト、既存 Garden コンポーネント活用

## 3. 画面仕様

### 3.1 ルーティング

```
/calendar              … デバイス自動判定でモバイルレイアウト切替
/calendar?force=pc     … モバイルでも PC レイアウト（緊急用、警告バナー表示）
```

### 3.2 モバイル レイアウト（縦持ち）

```
+--------------------------------+
| ≡  Garden Calendar         🔍 |  ヘッダー
+--------------------------------+
| 月 | 週 | 日 | リスト          |  ビュー切替（タブ）
+--------------------------------+
|                                |
|     カレンダー本体             |
|     （現在ビュー）              |
|                                |
+--------------------------------+
| 編集は PC で → リンク          |  フッター固定
+--------------------------------+
```

### 3.3 ビュー別の挙動

| ビュー | 挙動 |
|---|---|
| 月 | セルあたり最大 2 件、それ以上は「+n」タップで日詳細 |
| 週 | スワイプで前後週、タップで日詳細 |
| 日 | タイムライン縦スクロール、タップで詳細 |
| リスト | 直近 7 日、デフォルトビュー（モバイル既定） |

### 3.4 イベント詳細（モーダル / ボトムシート）

```

│ 営業予定                 │
│ A社 訪問                 │

│ 👥 山田、鈴木             │
│                         │
│ [PC で編集]  [閉じる]     │

```

## 4. デバイス判定ロジック

```ts
// useDeviceMode.ts 疑似コード
function detectDevice() {
  const ua = navigator.userAgent;
  const isMobileUA = /iPhone|iPad|Android|Mobile/.test(ua);
  const isNarrow = window.matchMedia('(max-width: 768px)').matches;
  return isMobileUA || isNarrow ? 'mobile' : 'desktop';
}
```

- SSR では User-Agent ヘッダで判定し、不一致時はクライアントで再判定
- `?force=pc` クエリで PC レイアウト強制（モバイルで横持ち + 拡大表示時の緊急用）

---

## 5. 編集禁止の徹底

### 5.1 UI レベル
- 編集 / 新規作成 / 削除ボタンを完全に非表示
- ドラッグ操作を disable
- イベント詳細の Edit リンクは「PC で編集」に置換

### 5.2 API レベル

```ts
// middleware.ts 疑似
if (isMobileUA(req) && isWriteEndpoint(pathname)) {
  return new Response('Mobile is read-only', { status: 403 });
}
```

書き込み系: POST / PATCH / DELETE on `/api/calendar/events*`

### 5.3 監査
- スマホからの read アクセスを `audit_log` に記録（user_id, ip, ua, accessed_at）

## 9. 判断保留事項

| # | 論点 | a-auto スタンス |
|---|---|---|
| 1 | スマホ閲覧の admin 承認方針 | 本 spec で承認 = 東海林さん（admin）が判断、ローンチ前に再確認 |
| 2 | タブレット（iPad 等）の扱い | width >= 768px は PC レイアウト、ただし read-only モードを維持する選択肢も |
| 3 | スマホでの編集要望が出たとき | v2 で限定スコープ（時刻調整のみ）を検討 |
| 4 | PWA 化 | v1 では行わない、v2 で評価 |
| 5 | 端末認証（生体 / PIN） | OS 任せ、Garden 側追加実装は v2 |
| 6 | force=pc の常時許可 | 警告バナー必須で許容、デバッグ用 |
| 7 | 通知タップでの起動 | C-06 とリンク、URL ディープリンクで対応 |

## 既知のリスクと対策

- **リスク 1**: 端末紛失時の情報漏洩 → セッション 24h 自動切断 + 二要素認証（既存）併用
- **リスク 2**: 横持ち時の UI 崩れ → 横向き専用メディアクエリ + 強制縦持ち推奨バナー
- **リスク 3**: User-Agent 偽装で書き込み API を叩く → API レイヤで二重チェック（UA + 端末認証トークン）
- **リスク 4**: スマホブラウザでの FullCalendar v6 動作不良 → list ビューを既定にし、month は dayGrid シンプル版で代替
- **リスク 5**: 「PC で編集」リンクの遷移先が分からない → ヘルプリンク併設
