## 概要

このドキュメントは、Garden-Bud（蕾）で使用するためにGarden-Root（根）に配置すべきマスタテーブルを定義したものです。

---

## ⚠️ 注意事項

### 1. マスタの原則

- **Garden-Rootはマスタの「唯一の情報源」**：Garden-Budはマスタを「参照するだけ」で「持たない」
- **マスタの更新はGarden-Rootで行う**：Garden-Budからマスタを直接編集しない
- **削除は厳禁**：マスタレコードは削除せず、無効フラグで管理する

### 2. 法人ごとの違いに注意

- 6法人それぞれ本店所在地・代表者が異なる
- 法人ごとにデフォルト振込銀行が異なる（5法人：楽天銀行、センターライズ：みずほ銀行）
- 従業員は複数法人に所属しない（1従業員＝1法人）

### 3. 給与関連の注意

- **雇用形態によって給与体系が異なる**：正社員/アルバイトで計算ルールが違う
- **業務委託は対象外**：Garden-Budの給与処理では扱わない（個別対応Excel）
- **一部別対応従業員あり**：給与体系マスタで個別設定できるようにする

### 4. 銀行口座の注意

- **法人ごとに別々の口座**：口座マスタは法人IDで紐づけ
- **振込先銀行は追加可能**：新しい銀行を追加したら、次回から全法人で選択可能にする
- **口座番号は7桁**：銀行CSVフォーマットに合わせる

### 5. キングオブタイム連携の注意

- **勤怠データはGarden-Rootに配置**：従業員に紐づく基礎データとしてマスタ側に置く
- **API連携で自動取込**：月次または毎日取込
- **従業員IDの紐づけが必要**：キングオブタイムの従業員IDとGarden-Rootの従業員IDを対応させる

### 6. MFクラウド給与との関係

- **MFクラウド給与は社労士用に継続利用**
- **従業員IDの対応表が必要**：MFクラウド給与の従業員番号とGarden-Rootの従業員IDを紐づける
- **給与明細発行はGarden-Budで行う**：MFクラウド給与の明細発行機能はOFF

## 2. 銀行口座マスタ

### 用途
- 法人ごとの振込元口座を管理
- 明細管理で入出金データを取り込む際の口座特定
- 振込管理で「どの口座から払うか」を選択

### フィールド定義

| フィールド名 | フィールドID | 型 | 必須 | 説明 |
|:--|:--|:--|:--|:--|
| 口座ID | account_id | 文字列（自動採番） | ○ | 例：ACC-001 |
| 法人ID | company_id | ルックアップ | ○ | 法人マスタから |
| 法人名 | company_name | 文字列 | - | 自動取得 |
| 銀行名 | bank_name | 文字列 | ○ | 例：楽天銀行 |
| 金融機関コード | bank_code | 文字列（4桁） | ○ | 例：0036 |
| 支店名 | branch_name | 文字列 | ○ | 例：第一営業支店 |
| 支店コード | branch_code | 文字列（3桁） | ○ | 例：251 |
| 口座種別 | account_type | 選択肢 | ○ | 普通/当座 |
| 口座番号 | account_number | 文字列（7桁） | ○ | 例：1234567 |
| 口座名義 | account_holder | 文字列 | ○ | 例：カ）ヒュアラン |
| 用途 | purpose | 選択肢 | - | メイン/給与/経費等 |
| 有効フラグ | is_active | チェック | ○ | デフォルト：ON |
| 作成日時 | created_at | 日時 | ○ | 自動 |
| 更新日時 | updated_at | 日時 | ○ | 自動 |

### 注意事項

- 1法人に複数口座を持てる（メイン口座、給与用口座など）
- 法人を選択すると、その法人の口座一覧が選択肢になる
- 新しい銀行を追加する場合は、このマスタにレコードを追加
