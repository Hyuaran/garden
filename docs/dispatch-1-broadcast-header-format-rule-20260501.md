## 投下用短文（東海林さんが各セッションにコピペ）

宛先部分「`a-XXXX`」を該当セッション名に置き換えてコピペしてください。例: `a-bloom` 宛なら「`a-XXXX`」→「`a-bloom`」。

~~~
#1 ⭐ 【a-main-010 から a-XXXX への 横断指示（dispatch ヘッダー形式 ルール改訂）】
発信日時: 2026-05-01(金) 23:30

dispatch / セッション間メッセージのヘッダー形式を恒久ルール化しました。
今後の発信ではこの形式に統一してください。

【標準ヘッダー形式】

#NNN <アイコン> 【<発信元> から <宛先> への <種別>（任意で 件名/topic）】
発信日時: YYYY-MM-DD(曜) HH:MM

その後、空行を 1 つ挟んで本文。

【発信日時の詳細フォーマット】

| 要素 | 形式 | 例 |
| 日付 | YYYY-MM-DD | 2026-05-01 |
| 曜日 | (月)(火)(水)(木)(金)(土)(日) | (金) |
| 時刻 | HH:MM（24 時間表記）| 23:30 |
| 完成形 | 半角スペース連結 | 2026-05-01(金) 23:30 |

→ 曜日と時刻まで必須。後追い時系列追跡のため。

【番号付与ルール (#NNN)】

- 採番は a-main セッションが管理（起草時に +1）
- a-main 系をまたいで通し番号（リセットなし、引き継ぎ書で次番号引き継ぎ）
- #1 から開始（過去発信は遡及対応せず、今後発信分から適用）
- 記載位置は【】の外側（前置き）

→ 東海林さんが「#114 OK」のような短い返答で識別可。

各セッションが a-main に発信する場合の番号は、a-main 側で再採番されます。
あなた側では番号付与不要、種別とアイコンのみ意識してください。
（a-main が受領時に番号を割り振り、a-main の管理 counter を更新）

【重要度アイコン】

| アイコン | 種別 | 用途 |
| 🔴 | 緊急 | 即対応必要、タイムクリティカル |
| ⭐ | 重要 / 優先 | 当日中の対応推奨 |
| 🟢 | 通常 | 通常優先度 |
| 🟡 | 中 | 緊急ではないが対応必要 |
| 📋 | 確認のみ | アクション不要、情報共有のみ |

【種別の語彙】

dispatch / 受領確認 / 進捗報告 / 完了報告 / 質問 / 確認 / 共有 /
横断指示 / 周知 / 引き継ぎ

【適用範囲】

- 他セッション向けの投下短文・dispatch・回答すべて
- 東海林さんがコピペで配布するすべてのメッセージ
- a-XXXX から a-main / 他モジュール宛の発信も対象（番号は a-main 側で付与）

【既存運用との整合】

ヘッダーは ~~~ or ``` のコードブロック内の冒頭 1 行目に必ず配置。
ファイル化 + markdown link 案内のルールはそのまま継続。
ファイル名にも番号付与推奨（dispatch-NNN-XXX-YYYYMMDD.md）。
