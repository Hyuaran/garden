- 本ファイルは a-root-002 セッションでの v3 ルール記録（公式 memory は a-main 側で管理）

## v3 標準ヘッダー形式（5 行構成、本文前に空行）

```
1 行目: <重要度アイコン> <接頭辞>-NNN
2 行目: 【<発信元> から <宛先> への <種別>(任意で 件名)】
3 行目: 発信日時 YYYY-MM-DD(曜) HH:MM
4 行目: 空行
5 行目以降: 本文
```

## 接頭辞ルール

各セッションごとに接頭辞 + カウンタを管理:

| セッション | 接頭辞 | カウンタファイル |
|---|---|---|
| a-root-002（本セッション） | `root-002` | `C:\garden\a-root-002\docs\dispatch-counter.txt` |
| 他セッション | （各 CLAUDE.md / 起動 dispatch で確認） | 各 worktree の `docs/dispatch-counter.txt` |

カウンタ初期: 2026-05-01 から 1 開始。発信ごとに +1。
カウンタファイルには **次に使う番号** を記録（"次番号メモ"）。

## 重要度アイコン（1 行目の番号の前に必須）

| アイコン | 重要度 | 用途 |
|---|---|---|
| 🔴 | 緊急 | 即時対応必要 |
| ⭐ | 重要・優先 | 当日中対応 |
| 🟢 | 通常 | 通常運用 |
| 🟡 | 中 | (アイコン定義あり、用途は文脈依存) |
| 📋 | 確認のみ | 確認だけ、対応不要 |

## 種別語彙

dispatch / 受領確認 / 進捗報告 / 完了報告 / 質問 / 確認 / 共有 / 横断指示 / 周知 / 引き継ぎ

## 発信日時フォーマット

`YYYY-MM-DD(曜) HH:MM`

- 例: `2026-05-01(金) 23:45`
- 曜日（月火水木金土日）と時刻まで必須
- 後追い時系列追跡のため

## ファイル名規則

dispatch をファイル化する場合:

```
docs/dispatches/dispatch-<接頭辞>-NNN-<内容>-<YYYYMMDD>.md
```

例: `docs/dispatches/dispatch-root-002-1-rule-v3-receipt-20260501.md`

## 引き継ぎ

a-root-002 → 後続セッション（a-root-003 等）への引き継ぎ時:

1. ハンドオフ書末尾に「**dispatch counter: 次番号 NNN**」を記載
2. 新セッション起動時、`docs/dispatch-counter.txt` に NNN を書き込み
3. ブランチ間でも同一カウンタを共有（worktree 単位）

## 発信例（a-root-002 → a-main-010）

```
🟢 root-002-1
【a-root-002 から a-main-010 への 完了報告(XXX)】
発信日時: 2026-05-01(金) 23:50

[本文]
```
