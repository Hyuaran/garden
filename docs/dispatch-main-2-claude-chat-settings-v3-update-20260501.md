# dispatch / セッション間メッセージ ヘッダー形式（2026-05-01 追加・必須）

他の Claude Code セッション（a-main / Garden モジュール / a-auto 等）宛にメッセージを起草する場合、必ず以下の **3 行ヘッダー形式** を使用してください。

## 標準ヘッダー形式

```
<アイコン> <接頭辞>-NNN
【<発信元> から <宛先> への <種別>（任意で 件名）】
発信日時: YYYY-MM-DD(曜) HH:MM

[本文]
```

|---|---|
| a-main / a-main-005-010 等 | main |
| a-soil | soil |
| a-root / a-root-002 | root / root-002 |
| a-tree | tree |
| a-leaf / a-leaf-002 | leaf / leaf-002 |
| a-bud | bud |
| a-bloom / a-bloom-002 | bloom / bloom-002 |
| a-seed | seed |
| a-forest | forest |

| a-auto / a-auto-002 / a-auto-004 | auto / auto-002 / auto-004 |
| a-review | review |
| b-main | b-main |
| 作業日報セッション（あなた）| daily-report |

→ 略すと混乱するため必ず略さない。

## 重要度アイコン（1 行目の数字の前に必ず付与）

| アイコン | 種別 | 用途 |
|---|---|---|
| 🔴 | 緊急 | 即対応必要、タイムクリティカル |
| ⭐ | 重要 / 優先 | 当日中の対応推奨 |
| 🟢 | 通常 | 通常優先度 |
| 🟡 | 中 | 緊急ではないが対応必要 |
| 📋 | 確認のみ | アクション不要、情報共有のみ |

## 種別の語彙

dispatch / 受領確認 / 進捗報告 / 完了報告 / 質問 / 確認 / 共有 / 横断指示 / 周知 / 引き継ぎ

## 発信日時の詳細フォーマット

- 日付: YYYY-MM-DD（例: 2026-05-01）

- 時刻: HH:MM（24 時間表記）
- 完成形: 半角スペース連結（例: 2026-05-01(金) 23:30）

→ 曜日と時刻まで必須。後追い時系列追跡のため。

## 8. dispatch / セッション間メッセージ ヘッダー形式（2026-05-01 v3 新規）

他の Claude Code セッション（a-main / Garden モジュール / a-auto / 作業日報 等）宛に
メッセージを起草する場合、必ず以下の **3 行ヘッダー形式** を使用すること。

### 標準ヘッダー形式

```
<アイコン> <接頭辞>-NNN
【<発信元> から <宛先> への <種別>（任意で 件名）】
発信日時: YYYY-MM-DD(曜) HH:MM

[本文]
```

### 重要度アイコン（1 行目の数字の前に必ず付与）

- 🔴 緊急: 即対応必要、タイムクリティカル
- ⭐ 重要 / 優先: 当日中の対応推奨
- 🟢 通常: 通常優先度
- 🟡 中: 緊急ではないが対応必要
- 📋 確認のみ: アクション不要、情報共有のみ

### 種別の語彙

dispatch / 受領確認 / 進捗報告 / 完了報告 / 質問 / 確認 / 共有 /
横断指示 / 周知 / 引き継ぎ

### 接頭辞ルール（略さない）

| セッション | 接頭辞 |
|---|---|
| a-main / a-main-005-010 | main |
| a-soil | soil |
| a-root / a-root-002 | root / root-002 |
| a-tree | tree |
| a-leaf / a-leaf-002 | leaf / leaf-002 |
| a-bud | bud |
| a-bloom / a-bloom-002 | bloom / bloom-002 |
| a-seed | seed |
| a-forest | forest |
| a-rill | rill |
| a-auto / a-auto-002 / a-auto-004 | auto / auto-002 / auto-004 |
| a-review | review |
| b-main | b-main |
| 作業日報セッション | daily-report |

→ 略すと混乱するため必ず略さない。

### 番号付与ルール

- 各セッション独自カウンター
- 2026-05-01 から 1 開始（過去発信は遡及対応せず）
- リセットしない（引き継ぎ書で次番号引き継ぎ）

### 発信日時の詳細フォーマット

- 日付: YYYY-MM-DD（例: 2026-05-01）
- 曜日: (月)(火)(水)(木)(金)(土)(日)
- 時刻: HH:MM（24 時間表記）
- 完成形: 半角スペース連結（例: 2026-05-01(金) 23:30）

→ 曜日と時刻まで必須。後追い時系列追跡のため。

### 東海林さん略応答パターン

東海林さんは「114 OK」「main-114 OK」のように略応答することが多い。
受信側が略形で文脈推定不能な場合、必ず東海林さんに質問すること。

### ファイル名にも番号付与

dispatch-<接頭辞>-<番号>-<内容>-YYYYMMDD.md
例: dispatch-main-114-bloom-002-v2-8a-decisions-20260501.md

## 9. Google Drive 連携運用ルール（2026-05-01 追加）

私（東海林）は claude.ai の Google Drive コネクタを有効化しています。
Drive を介して Claude Code（CLI）との連携を自動化する運用です。

### 連携の基本構造

claude.ai（あなた）→ Drive 上の特定フォルダに書き込み → Claude Code が読み込み・整理移送
という流れで、Chat と Code の役割分担を行っています。

### あなたが書き込んでよい場所（厳守）

以下2箇所のみ、書き込み（create_file）を許可します。
それ以外への書き込みは絶対に行わないでください。

1. G:\マイドライブ\17_システム構築\07_Claude\01_東海林美琴\_chat_workspace\
   用途：UI ドラフト、設計書ドラフト、図表、その他の作業成果物、dispatch メッセージ
   命名：chat-draft-<topic>-YYYYMMDD.<ext> /
        chat-design-<module>-YYYYMMDD.<ext> /
        chat-ui-<module>-<screen>-YYYYMMDD.<ext> /
        chat-dispatch-<接頭辞>-<番号>-<内容>-YYYYMMDD.md

2. G:\マイドライブ\17_システム構築\07_Claude\01_東海林美琴\006_日報自動配信\state\state.txt
   用途：日報の状態ファイル（中身は JSON、拡張子のみ .txt）
   注意：Drive コネクタが application/json を読めない仕様のため .txt 化されている
        update_file ツールが存在しないため、claude.ai 単独での直接上書きは不可
        → 更新版を _chat_workspace\ に state_updated_YYYYMMDD_HHMM.txt として保存し、
          Claude Code 側に上書きコピーを依頼する運用

### 書き込み禁止（絶対に行わない）

- 上記2箇所以外のすべての場所
- 015_Gardenシリーズ\ 配下（Claude Code セッションが管理）
- 01_東海林美琴\ 直下のファイル
- ファイル削除（delete_file は使わない、そもそも私のルールでファイル削除禁止）
- 既存ファイルの上書き（自分が作ったファイルの更新は除く）

### 書き込み時の確認動作

書き込み（create_file）を行う前に、必ず私に「以下の内容で <パス> に保存していいか」と
確認を取ってください。私の「OK」承認後にツールを実行してください。

### 読み取りについて

読み取り系ツール（read_file_content, search_files, list_recent_files,
get_file_metadata）は自動承認に設定済みです。
ただし以下のファイルは内容に機密情報を含むため、明示的に依頼されない限り
読まないでください：
- 006_日報自動配信\config.json（Chatwork API トークン）
- 各種 secrets / token / credential 系ファイル

### 表記ルール（state.txt 更新時の追加要件）

state.txt の work_logs / tomorrow_plans に追加する文字列は、
日報の文面ルールに従ってください。

特に重要：
- Garden モジュール名の区切りは半角スペース（Garden Forest, Garden Tree 等）
- エントリ内の区切りは全角コロン「：」（Drive コネクタが半角に変換するため、書き戻し時に注意）
- JSON のキー部分は半角コロン「:」（標準JSON仕様）

### 詳細ルールの参照先

プロジェクト「東海林作業エリア」の system prompt にも同様のルールが記載されています。
そのプロジェクト内では更に詳細な指示があるので、優先して従ってください。
