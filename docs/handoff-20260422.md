### 🔴 重要な設計判断（supabase-py → requests への切替）
- 当初 `supabase-py` を使う予定だったが、**Python 3.14 で依存 `pyiceberg` のビルド失敗**（MS C++ Build Tools 必須）
- `requests` で Supabase REST API を直接叩く方式に変更済み
- 実装プランも `requests` ベースに更新済み
- REST API の叩き方:
  ```python
  endpoint = f"{url}/rest/v1/shinkouki"
  headers = {
      "apikey": service_key,
      "Authorization": f"Bearer {service_key}",
      "Content-Type": "application/json",
      "Prefer": "return=representation",
  }
  requests.patch(endpoint, headers=headers, params={"company_id": "eq.hyuaran"}, json=payload)
  ```
