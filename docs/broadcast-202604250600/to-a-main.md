## Batch 14 の独自ハイライト

### 1. 改ざん不可能な記録領域
RLS で UPDATE/DELETE 完全禁止、admin/super_admin でも改ざん不可。Trigger 経由 INSERT のみ許可。

### 2. Kintone 改善案
Kintone の削除済管理は同テーブルに残るため検索負荷集中 → Garden では `deleted_at IS NULL` を UI 側判定で分離、削除済バッジで可視化。

### 3. PII 暗号化 + 復号監査
機密列（マイナンバー / 銀行口座等）は pgcrypto で暗号化、admin+ のみ復号関数経由でアクセス、復号も全件監査。

### 4. 4 段階展開計画
基盤（Phase 1）→ 軽量モジュール（Phase 2）→ 中規模（Phase 3）→ 大型（Phase 4）の慎重展開、各 2 週間検証期間。

### 5. UNDO snackbar 5 秒
削除直後 5 秒以内なら 1 クリック復元、誤操作の即時取消を実現（自分の削除のみ）。

### 6. 4.4 億行への対応
7 年累計想定 4.4 億行に対し、pg_partman 月次パーティションを Phase 1 から導入、検索性能維持。
