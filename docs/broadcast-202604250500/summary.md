## 🔑 各 spec の核心

### #01 画像検索 🔴
- **8 条件複合検索**（PD / お客様番号 / 供給地点 / 電話 / 住所 / リスト名 / 申込者名 / 代表者名）
- pg_trgm GIN インデックスで部分一致高速化
- 2 サイズサムネ事前生成（200×200 / 800×800）
- 削除済バッジ可視化（誤削除認識用）
- レスポンシブグリッド（mobile 2 列 → desktop 6 列）
- 253 万件規模で 1 条件 < 50ms / 8 条件 AND < 500ms

### #02 スマホ専用ホーム 🔴
- **中央円グラフ**（月間件数、4 弦ステータス分割）
- 周囲 6 アイコン配置（撮影 / 検索 / 新規 / 一覧 / KPI / 設定）
- 上タブ 3 つ（マイページ / 案件一覧 / ホーム）
- < 768px 専用、デスクトップとは独立した UI
- カメラ直接起動 + Canvas 圧縮 + オフライン耐性

### #03 レスポンシブ対応 🔴
- 3 層（Mobile < 768 / Tablet 768-1280 / Desktop > 1280）
- 既存 8 画面の点検と移行方針
- 5 レイアウトパターン（グリッド / テーブル → カード / モーダル → フルスクリーン / フォーム / タブ）
- タッチ vs マウス分岐（hover 効果切替）
- 5 端末以上での実機検証

### #04 質問機能 + 関電 Excel 🔴
- 案件ごとの質問・回答履歴（カテゴリ 6 種、緊急度 4 段階）
- 関電キャリア対応依頼 Excel **取込**（自動マッチング 顧客番号 > 供給地点 > 名前+住所）
- 関電指定フォーマットで **回答出力**
- 期限管理 + 3 日前リマインダー Cron
- バッチ（同一 Excel）単位の管理ダッシュボード

### #05 ログイン拡張 🟡
- ログイン時の位置情報自動取得（Geolocation API）
- 1 日 1 回の稼働場所申告（大阪本社 / 滋賀支店 / 在宅 / 営業先）
- グループライン自動送信（既存 Chatwork 経由、手動送信を自動化）
- ログイン後ロードマップポップアップ（Tree 風、担当案件 + 関電依頼 + 優先タスク）
- 「今日は表示しない」localStorage 制御

### #06 マイページ拡張 🟡
- 6 業務支援項目追加:
  - 検針カレンダー（地域別、年初取込）
  - 燃調費一覧（月次更新）
  - 市場価格（JEPX、Phase 2 で Cron 自動取込）
  - 関電チャットリンク
  - 関電 FD（PDF 資料 + 外部 URL）
  - 他社電力会社一覧
- カード形式 + admin データ管理 UI
- データソース別の更新方針明記

### #07 新規送信 UI 🟡
- 2 カラムレイアウト（左=奪還 / 右=囲込）
- 電気ガスセット契約フォーム（contract_type 拡張）
- 送付状況管理（追跡番号付き）
- 過去履歴自動検出（同電話番号、1 年以内）
- カラム間コピー機能（誤入力対策）

### #08 営業リスト解放管理 🟡
- 期マスタ（4 ヶ月単位、3 期/年）
- リスト解放制御（admin、部分解放可）
- アポ禁マスタ（期跨ぎ管理、自動判定）
- **期切替バッチ**（前期見込みアポ → 新期アポ禁に一括登録）
- 営業画面で解放リストのみ表示、架電前にアポ禁判定

## 🚨 重要リスクと対策

### R1: 関電 Excel フォーマット未確定
- **対策**: M3 開始前に関電キャリアと協議、サンプル Excel 入手
- **判定**: フォーマット仕様書取得 + α版で実証

### R2: 大量データ（253 万件）の検索性能
- **対策**: pg_trgm GIN + 部分インデックス + サムネ事前生成
- **判定**: 1 条件 < 50ms / 8 条件 < 500ms 計測

### R3: モバイル UX 品質
- **対策**: 5 端末以上で実機検証、Lighthouse Mobile スコア 90+
- **判定**: iPhone / Android / iPad で 60fps 維持

### R4: 関電業務との整合性ミス
- **対策**: α版で東海林さん 1 人、β版で関電責任者立会い
- **判定**: 現場責任者の最終承認

### R5: 削除済データの誤判断
- **対策**: 削除済バッジ可視化 + フィルター（既定: 削除済含む）
- **判定**: 全画面で削除済の視認性確認
