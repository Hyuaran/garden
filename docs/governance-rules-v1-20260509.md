## 0. 本書の位置付け

本書は a-main セッション運用に関する **合意事項を独立保存**するための governance ドキュメント。
- handoff 文書に分散していた決定事項を 1 か所に統合
- 次回 main 引越し時、handoff だけでなく本書も必ず参照
- ルール改訂時は本書を更新 + 三点セット同期

## 1. ガンガンモード本質定義

memory `feedback_gangan_mode_default` v2 に明示。

| 軸 | 内容 |
|---|---|
| 1 | **全モジュール並列稼働**（main 指示で全セッション継続稼働、1 経路集中禁止）|
| 2 | **5h 枠フル活用**（残量見て手を止めない）|
| 3 | **東海林作業時間無視**（投下作業負荷を懸念して dispatch 数を絞らない）|

派生効果（手段論）: 確認過多回避、即実行連鎖、preview tool 視覚確認。

「通常モード」「待機モード」表現はガンガン解除ではない（運用状態を指す）。

## 8. 厳しい目で再確認 N ラウンド（焦点別 + 連続 3 ラウンド 0 件で終了）

memory `feedback_strict_recheck_iteration` 準拠。

| ラウンド | 焦点 |
|---|---|
| 1 | 対処済か |
| 2 | 対処の質（トリガー / 具体手順 / 例外）|
| 3 | メタレベル抜け（自己評価の甘さ / 仕組み自体の抜け）|
| 4 | 実装漏れ（実際に動くか / テンプレ落とし込み / 日常呼び出し可能か）|
| 5 | 矛盾検出（既存ルール矛盾 / 二重管理 / 状態認識）|
| 6+ | 必要なら継続、新しい角度 |

**終了条件**: 連続 3 ラウンド抜け 0 件 + 私の盲点リスト宣言 + 東海林さん最終チェック（B+C ハイブリッド）。

**自発トリガー**: 東海林さんへの全提案 / 全報告前に**私が自発で発動**。促されてからでは遅い。
