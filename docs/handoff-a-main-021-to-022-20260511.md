### Tree D-01 apply エラー（最重要）

```
SQL クエリの実行に失敗しました:
エラー: 42830: 参照テーブル「root_employees」に対して、指定されたキーに一致する一意制約がありません
```

| 観点 | 詳細 |
|---|---|
| エラーコード | PostgreSQL 42830 = invalid_foreign_key |
| 真因 | Tree D-01 schema の FK 参照先 `root_employees.<col>` に UNIQUE 制約が欠落 |
| 推測 | `employee_number` or `user_id` を FK 参照 → そのカラムに UNIQUE 制約必要、root_employees 側に未設定 |
| 対応 | a-root-002 連携、root_employees に UNIQUE 制約追加 migration 起草 → 再 apply |
| ロールバック | PostgreSQL transaction 単位で自動 rollback 想定、新規テーブル作成途中残らない（要確認、`SELECT proname FROM pg_proc WHERE proname LIKE 'tree_%'` 等で確認）|
| dispatch | a-tree-002 + a-root-002 連携 dispatch 起草（a-main-022 で）|
