### 本文（v5.2）

#### ルール

dispatch 提示は以下 v5.2 形式を厳守:

| 行 | 内容 |
|---|---|
| 1 | 🚨 dispatch 発行 🚨（または 🚨🚨🚨 PowerShell 利用 🚨🚨🚨）+ 緊急度マーク |
| 2 | 空行 |
| 3 | 投下先: ... |
| 4 | ファイル: [filename](path) ← markdown link 必須 |
| 5 | 別途添付必要データ: なし / あり（あり時はフルパス記載）|
| 6 | 空行 |
| 7- | dispatch ヘッダー（投下用 ~~~ ラップ内に冒頭 3 行を配置）|
| 末尾 | → 投下完了したら「NNN投下済」で OK |

コピペ本体（dispatch 全文）はチャット内に出さない。東海林さんは markdown link をクリックして md ファイルを開き、そこからコピーする。

#### Why

- 5/9 18:00 東海林さん指摘: 「投下用短文を通常文章として記載 + コピペテキストだらだら出力」がわかりにくい
- 016 期: dispatch 全文を 2 重に表示（説明 + コピペ本体）= 東海林さんが「どこからどこまでコピーすべきか」判断必要
- ガンガンモードで多投時、dispatch 本文が埋もれて投下漏れリスク
- markdown link 化で Claude Code 内クリック開閉可能（システム指示準拠）
- 5/11 a-audit-001 audit-001- No. 2 指摘: v5.1 では「~~~ 内に ` ``` ` 禁止」のみ明記、~~~ 内 ~~~ ネスト + 冒頭 3 行 ~~~ 外配置が運用上未抑止

#### How to apply

##### 単一 dispatch の場合

外ラップ内に通常 markdown（# / ## / 表 / 箇条書き / 引用）のみ。バッククォート 3 個 / ~~~ 自身のネスト禁止。冒頭 3 行は ~~~ 内配置必須。

##### PowerShell 系の場合

🚨🚨🚨 マークで強い注意喚起。実行コマンド + 影響範囲 + ファイル参照を冒頭明示。

##### 複数 dispatch 同時発行（ガンガン投下時、最重要）

🚨 dispatch 発行 (N 件、順不同で OK) 🚨 + # 1 / # 2 / # 3 リスト化。投下完了は各番号で順次返信。

#### md ファイル側の dispatch ヘッダー

md ファイル本体には従来通り（v5.2）の dispatch ヘッダー + 投下用 ~~~ ラップを記載。投下用 ~~~ ラップ内に冒頭 3 行（番号 + 元→宛先 + 発信日時）を配置、~~~ 外に「詳細（参考、投下対象外）」セクション。

#### 投下用短文 ~~~ 内の禁止事項（v5.2、最重要）

投下用短文 ~~~ ラップ内に **同記号繰返しブロック（~~~ / バッククォート 3 個）を入れることを禁止**（コピペ分断防止）。

| 違反パターン | 規定 | 起源 |
|---|---|---|
| 違反 7（v5.1 既往）: ~~~ 内にコードブロック（バッククォート 3 個） | 禁止 | 5/9 20:30 東海林さん指摘 |
| 違反 7-b（v5.2 拡張）: ~~~ 内に ~~~ 自身のネスト | 禁止 | 5/11 a-audit-001 audit-001- No. 2 |
| 違反 9（v5.2 新規）: 冒頭 3 行（番号 + 元→宛先 + 発信日時）を ~~~ 外配置 | 禁止、冒頭 3 行は ~~~ 内配置必須 | 5/11 a-audit-001 audit-001- No. 2 |

##### 理由

- Markdown レンダラーが ~~~ / ` ``` ` を別ブロックと認識し、コピー時にテキストが分断される
- 東海林さんがコピペで「~~~」全範囲をコピーしたつもりが、内部の同記号で切れる
- ~~~ 自身のネスト = レンダラーが内側の ~~~ で外側ラップを閉じてしまう（最も致命的）

##### 対処

| 内容 | 旧（NG）| 新（OK）|
|---|---|---|
| 完走報告フォーマット例 | ~~~ 内に ~~~ ラップ or ` ``` ` でコードブロック化 | ~~~ 外（詳細セクション）に移動 + 引用ブロック（`>`）や箇条書き、または別 md ファイル化 |
| HTML / コード例 | 同上 | 同上 |
| 期待応答フォーマット例 | 同上 | 同上 |

##### 雛形ベストプラクティス（v5.2 新規）

dispatch 内に報告フォーマット雛形（audit-001- No. NN / analysis-001- No. NN 等）を提示する場合、~~~ ネスト + コードブロックは使わず以下方式で記述する:

| 方法 | 内容 |
|---|---|
| ### サブヘッダーで構造を示す | 「### 件名」「### 内容」「### self-check」等で雛形 outline を提示 |
| 例示部分は通常 markdown 記述 | 表 / 箇条書き / 引用ブロック（`>` で囲む）|
| HTML / コード例を示す必要がある場合 | 外ラップ ~~~ を撤廃 → 通常 markdown 文書として提示、または別 md ファイル化（dispatch 本文と分離）|

##### OK パターン例（v5.2）

外ラップ ~~~ の中で、冒頭 3 行 + 件名 + 内容 + 期待応答形式（### サブヘッダー）+ self-check を通常 markdown で記述。コードブロック / ~~~ ネスト一切使わない。

##### 既存 dispatch の修正方針

| 状況 | 動作 |
|---|---|
| 投下未済 dispatch | 投下前に v5.2 修正 |
| 投下済 dispatch | 過去分は触らない、本ルールは新規起草分から適用 |
