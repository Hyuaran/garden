## 注意点・詰まっている点

### ⚠️ A-03 SQL の FK 型修正済み
A-03 初版は `bud_transfers(id uuid)` を参照していたが、実 PK は `transfer_id text`。
SQL ファイルは修正済み（commit `de45013` 参照）。
**初版 SQL を東海林さんが既に適用済みの場合は、テーブル DROP + 再適用が必要**。

### ⚠️ A-06 spec との実装差分
- spec の `matched_transfer_id uuid` → 実装 `text`（実 PK 整合性）
- spec の `bud_is_user()` / `bud_has_role(...)` は不存在 → 既存 `bud_has_access()` / `bud_is_approver_or_above()` / `bud_is_admin()` を使用
- spec の auto-match SQL クエリは TS の `bulkMatch` で代替（純関数化でテスト容易）

### ⚠️ Chatwork 環境変数未設定時の挙動
- `CHATWORK_API_TOKEN` / `CHATWORK_BUD_ROOM_ID` 未設定なら**通知スキップ**（console.warn のみ）
- メイン処理（ステータス遷移）は通知失敗でも完了
