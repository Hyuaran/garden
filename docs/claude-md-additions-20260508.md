## 追加版（§20 + §21 + §22 新規）

### § 20. Claude 使用環境（2026-05-08 追加）

#### 20-1. 使用ツール

| ツール | 形態 | 用途 |
|---|---|---|
| **Claude Code** | デスクトップアプリ版 | Garden 開発・dispatch 起草・コードレビュー（メイン）|
| **Claude.ai** | WEB ページ（Chrome 等で常時開いている）| デザインレビュー / chat ベースの相談 / claude.ai 起草版 UI 制作 |

#### 20-2. 連携運用

- Claude Code = ファイル / git / コード操作主体（CLI 系作業）
- Claude.ai = ブラウザベースの設計議論 + UI 起草（HTML 試作版）
- 両者の連携: `_chat_workspace/` 配下で markdown / HTML 経由で相互参照
- 例: claude.ai 起草版 UI → `_chat_workspace/garden-bud/ui_drafts/*.html` → Claude Code で `015_Gardenシリーズ/000_GardenUI_<module>/` に整理移送

#### 20-3. 操作の前提

- Claude Code は **デスクトップアプリ版** = ファイル / git / Bash / settings.json 直接操作可
- Claude.ai は **ブラウザ環境** = ファイル直接操作不可、artifact / canvas 中心

### § 21. コンテキスト 50% 超過アラート（2026-05-08 追加）

#### 21-1. 目的

セッションのコンテキスト使用率が 50% を超えたら、**担当セッション内で東海林さんに自動通知**することで、引っ越し準備の余裕を確保する。

#### 21-2. アラート発動条件

| 使用率 | アクション |
|---|---|
| 50% 到達 | **アラート 1 回**: 「コンテキスト 50% 到達、引っ越し準備推奨」を担当セッション内で表示 |
| 60% 到達 | **アラート 2 回目**: 「引っ越し開始推奨」+ §22 引っ越しルール案内 |
| 70% 到達 | **アラート 3 回目**: 「引っ越し必須」+ §22 即実行 |
| 80% 到達 | 既存ルール（CLAUDE.md §「使用率別アクション」）= 完全ハンドオフメモ書き出し + push |

#### 21-3. アラートの表現

```
⚠️ コンテキスト使用率 [50/60/70]% 到達
推奨アクション: [引っ越し準備 / 引っ越し開始 / 引っ越し必須]
詳細: §22 セッション引っ越しルール参照
```

#### 21-4. 確認方法

- Claude Code の `/cost` または `/context` コマンドで使用率確認可能
- 各セッションは作業の節目（ファイル編集完了後 / 重い処理後）で確認
- 自動 hook で 50% 到達検知できる場合は hook 設定検討（PreCompact / SessionEnd 等）

### § 22. セッション引っ越しルール（2026-05-08 追加）

#### 22-1. 引っ越しタイミング

| 使用率 | 引っ越し |
|---|---|
| 60% 到達 | 引っ越し**準備**（フォルダ作成 + ハンドオフメモ起草）|
| 70% 到達 | 引っ越し**実行**（次セッション起動 + コピペテキスト発行）|
| 80% 到達 | 引っ越し**完了必須**（強制終了直前、最後のハンドオフメモ書き出し + push）|

#### 22-2. 引っ越し時の準備（Claude Code が実施）

新セッション用フォルダ作成は **Claude Code** が自動実行する:

```bash
# 例: a-main-014 → a-main-015 引っ越し
git worktree add C:/garden/a-main-015 -b workspace/a-main-015 origin/develop
# または既存ブランチ派生
```

#### 22-3. 次セッションへの引き継ぎコピペテキスト発行

引っ越し時、現セッションが **次セッションへの起動プロンプト**を生成し、東海林さんがコピペで起動可能にする:

```
[次セッション名] として作業を引き継いでください。

以下を順番に実施:
1. pwd で C:\garden\[新セッションパス] 確認
2. git status で workspace/[新ブランチ] ブランチ確認
3. git pull origin workspace/[旧ブランチ] で最新を取得（handoff 含む）
4. docs/handoff-[旧→新]-YYYYMMDD.md を精読
5. dispatch counter 確認
6. 累計成果と現在進行中タスクの把握
7. 待機中ジョブ確認
8. ★最初のタスク: [具体的タスク]
9. 完了したら「[新セッション名] 起動完了」と返答

注意点:
- ガンガン常態モードは恒久（東海林さん明言）
- ...
```

→ 東海林さんは新セッション起動 + このテキストを 1 回コピペするだけで引き継ぎ完了。

#### 22-4. メモリー忘れ確認（現セッションが実施）

引っ越し前、現セッションは **メモリー忘れがないか確認**する:

| 確認項目 | 内容 |
|---|---|
| handoff ファイル | `docs/handoff-[旧→新]-YYYYMMDD.md` に **全進行中タスク + 完走済タスク**を記載 |
| dispatch counter | 最新番号を handoff に記載 |
| 待機中ジョブ | 各モジュールセッションの完走報告待機状態を一覧化 |
| 判断保留事項 | 東海林さん判断仰ぎ未回答事項を一覧化 |
| 関連ファイル | 重要 doc（spec / plan / dispatch）への link を handoff に集約 |
| memory（feedback / project / user）| 直近セッションで追加・更新したものがあれば handoff に明示 |

→ ハンドオフメモのテンプレートは CLAUDE.md §「ハンドオフメモのテンプレート」参照（既存）。

#### 22-5. 引っ越し完了後の旧セッション

- 旧セッションは「引っ越し完了、新セッションへ移行」を表示して停止
- 旧セッション内で新規作業は禁止（push 衝突防止）
- 旧セッションのコンテキストは保持（再起動不要）= 振り返り用に残す

#### 22-6. 例外: ガンガン常態モード継続中の引っ越し

- 引っ越し中も他セッション（a-bud / a-soil 等）は継続稼働 OK
- a-main 系統のみ引っ越し対象
- モジュールセッションの引っ越しは個別判断（80% 到達時のみ）
