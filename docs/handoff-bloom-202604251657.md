## 2. 自走レビュー時の判断フロー

新規 PR 通知（または明示的レビュー依頼）受領時：

1. **対象判定**: PR title / body / files から事前準備チェックリストにマッチする領域を特定
2. **既存 spec 整合チェック**:
   - 横断 spec (PR #25 cross-cutting 6 件)
   - known-pitfalls.md 3 pitfall（timestamptz 空文字 / RLS Route Handler / 空オブジェクト insert）
   - 該当モジュールの spec / plan
3. **コード PR の場合の追加チェック**:
   - 型・テスト・lint 通過確認
   - main / develop への直接 push でないこと（feature ブランチ → develop PR 順守）
   - src/ への影響範囲を git diff で確認
4. **判定**:
   - 🟢 approve 候補 → コメント投稿（approve 判断は東海林さん帰宅後）
   - 🟡 軽微指摘 → コメント投稿
   - 🔴 重大問題 → コメント投稿のみ、approve しない、即停止 + pause file 作成

### 即停止条件（pause file 作成: `docs/pause-YYYYMMDDHHMM-a-bloom.md`）
- レビュー観点が事前準備チェックリストにない新パターン発見
- 設計判断が必要な指摘
- spec / plan 自体に矛盾発見
