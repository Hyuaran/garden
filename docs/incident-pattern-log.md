| 重大度 | 定義 | 動作 |

| 🟢 軽微 | typo / 表現微調整 / 軽微な状態認識ズレ | log 記録のみ、main へ「OK 流す」確認返答 |

## §4 類似パターン検出ルール

新規違反発生時、a-audit は以下の手順で類似性を検出:

### 4-1 検出フロー

1. 新規違反内容を §2 §3 baseline と照合
2. 「該当 memory」「真因」「検出シグナル」3 軸で類似性判定
3. 類似度 score（0-100）を算出（軸一致 = 33.3 点 × 一致軸数）
4. 類似度 ≥ 67%（2 軸以上一致）→「類似パターン」として main に報告
5. 類似度 < 67% → 新規パターンとして本 log に追加

### 4-2 検出シグナル一覧（新規違反検出用ヒューリスティクス）

| カテゴリ | シグナル |
|---|---|
| 既存実装把握 | dispatch 起草前 grep / Read / git log / dispatch counter / 直近 5 dispatch 履歴の未参照 |
| 外部依頼前 | mock 画像 / 参照ファイル / spec の Read 確認未実施 |
| 状態認識 | 応答冒頭 [一時停止中]/[稼働中] 未明示、状態と実行内容の不一致 |
| 自己評価 | 「全部対処済」「OK」発言時の N ラウンド未実施 / 東海林さん最終チェック未依頼 |
| 起動時症状 | counter 値 / 必読 docs / handoff 引継ぎ項目の未確認 |
| 三点セット同期 | CC memory 更新と claude.ai snapshot 更新の片側のみ実施 |
| ファイル参照 | Read で「OK」判定時に ls 物理存在検証未実施 |
| 提案数上限 | 4 択以上の選択肢提示 |
| dispatch 形式 | 冒頭 3 行（番号 + 元→宛先 + 発信日時）抜け、~~~ ラップなし、自然会話形式混入、**外ラップ ~~~ 内の同記号繰返しブロック有無（~~~ / ` ``` `）**（2026-05-10 違反 7 受け追加、main- No. 209 東海林さん GO 済）|
| memory 編集権限 | a-analysis / a-audit による memory 直接編集（権限違反）|
| 検出値根拠 | critique 内の数値報告で集計方法（機械集計 / 目視 / 概算）の明記なし、結論を左右する数値が手動目視のみ（2026-05-10 違反 8 受け追加）|

### 4-3 重大度判定基準

| 観点 | 🔴 重大 | 🟡 中規模 | 🟢 軽微 |
|---|---|---|---|
| 業務影響 | 不要再着手 / データ損失リスク | 1 サイクル遅延 | 影響なし |
| セッション信頼性 | リセット症状（複数違反併発）| 単発違反 | 表現ズレ |
| 再発リスク | 構造的（仕組み欠如）| 個別運用ミス | 偶発 |
| 東海林さん負荷 | エスカレーション必須 | main 経由報告 | log 記録のみ |
