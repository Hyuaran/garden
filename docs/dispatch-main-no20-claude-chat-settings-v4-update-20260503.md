# Queue Processor v2 仕様（2026-05-03 更新）

state.txt の更新は、以下のフローで自動反映されます：

1. あなた（claude.ai）が _chat_workspace\state_updated_YYYYMMDD_HHMM.txt として保存
2. 14:15 / 22:15 の自動送信直前 + 手動 send_report.py 実行時に send_report.py が _sync_pending() を呼んで反映
3. バックグラウンド処理（タスクスケジューラ「日報Queue処理」）は撤去済（2026-05-03 事故対応で C 案リファクタ）
4. 反映タイミングは 14:15 / 22:15 の送信時点 or 手動実行時

つまり、あなたは _chat_workspace\state_updated_*.txt として保存するだけで OK。
バックグラウンドでの自動反映ではなく、send_report.py 実行時に反映される仕様です。

## 8. dispatch / セッション間メッセージ ヘッダー形式（v4 確定 2026-05-02）

他の Claude Code セッション（a-main / Garden モジュール / a-auto / 作業日報 等）宛に
メッセージを起草する場合、必ず以下の **3 行ヘッダー形式（v4）** を使用すること。

### 標準ヘッダー形式（v4: 「No. NNN」付き）

```
<アイコン> <接頭辞>- No. NNN
【<発信元> から <宛先> への <種別>（任意で 件名）】
発信日時: YYYY-MM-DD(曜) HH:MM

[本文]
```

### v4 改訂理由

- 旧形式「main-9」だと a-main-010 と紛らわしい
- 新形式「main- No. 9」で「-」と「No. 」を挟むことでセッション名と明確に区別
- 「report-1」（旧 daily-report-1）も「report- No. 1」に変更

### 重要度アイコン（1 行目の数字の前に必ず付与）

- 🔴 緊急: 即対応必要、タイムクリティカル
- ⭐ 重要 / 優先: 当日中の対応推奨
- 🟢 通常: 通常優先度
- 🟡 中: 緊急ではないが対応必要
- 📋 確認のみ: アクション不要、情報共有のみ

### 種別の語彙

dispatch / 受領確認 / 進捗報告 / 完了報告 / 質問 / 確認 / 共有 /
横断指示 / 周知 / 引き継ぎ

### 接頭辞ルール（略さない、v4 で report に短縮）

| セッション | 接頭辞 |

- リセットしない（引き継ぎ書で次番号引き継ぎ）

- 曜日: (月)(火)(水)(木)(金)(土)(日)
